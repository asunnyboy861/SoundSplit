### 无直接可二次开发项目的原因

iOS音量控制领域在GitHub上几乎没有开源项目，原因如下：
1. iOS沙盒限制使得系统级音量控制无法通过常规App实现
2. 音量管理类App通常为商业项目，不开源
3. 核心技术依赖Apple私有框架，开源风险高

### 核心技术实现方案

#### 方案概述：Volume Profile + 自动化切换

由于iOS不允许第三方App直接修改系统通知音量，SoundSplit采用以下等效方案：

```
核心思路:
1. Volume Profile（音量配置文件）
   → 预设不同场景的音量组合
   → 一键切换所有音量设置

2. Shortcuts + AppIntents（自动化）
   → 通过Shortcuts自动切换Profile
   → 支持时间触发、位置触发、NFC触发

3. Focus Mode集成（场景自动切换）
   → Work Focus = 办公室Profile
   → Sleep Focus = 睡眠Profile
   → Personal Focus = 个人Profile

4. 自定义通知音量（音频预渲染）
   → 为不同音量级别预渲染通知音文件
   → 通过UNNotificationSound指定不同音量的声音文件

5. Per-App Volume（应用级音量）
   → 检测前台App自动切换Profile
   → 打开游戏时自动切换到"游戏"Profile
```

##### 2. 音频引擎管理器

```swift
import AVFoundation

final class AudioEngineManager: ObservableObject {
    
    static let shared = AudioEngineManager()
    
    private let audioSession = AVAudioSession.sharedInstance()
    
    @Published var currentRingerVolume: Float = 0.8
    @Published var currentMediaVolume: Float = 0.5
    
    enum VolumeCategory {
        case ringerAndAlerts
        case media
    }
    
    func applyProfile(_ profile: VolumeProfile) {
        setRingerVolume(profile.ringerVolume)
        setMediaVolume(profile.mediaVolume)
        currentRingerVolume = profile.ringerVolume
        currentMediaVolume = profile.mediaVolume
    }
    
    func setRingerVolume(_ volume: Float) {
        let clampedVolume = max(0.0, min(1.0, volume))
        do {
            try audioSession.setCategory(.playback, mode: .default, options: [.mixWithOthers])
            try audioSession.setActive(true)
            
            let player = try createSilentPlayer()
            player.volume = clampedVolume
            player.play()
            
            DispatchQueue.main.asyncAfter(deadline: .now() + 0.1) {
                player.stop()
                try? self.audioSession.setActive(false, options: .notifyOthersOnDeactivation)
            }
        } catch {
            print("Failed to set ringer volume: \(error)")
        }
    }
    
    func setMediaVolume(_ volume: Float) {
        let clampedVolume = max(0.0, min(1.0, volume))
        do {
            try audioSession.setCategory(.ambient, mode: .default)
            try audioSession.setActive(true)
            
            MPVolumeView.setVolume(clampedVolume)
        } catch {
            print("Failed to set media volume: \(error)")
        }
    }
    
    private func createSilentPlayer() throws -> AVAudioPlayer {
        guard let url = Bundle.main.url(forResource: "silence", withExtension: "caf") else {
            throw AudioError.silentFileNotFound
        }
        return try AVAudioPlayer(contentsOf: url)
    }
    
    func getCurrentSystemVolume() -> Float {
        return audioSession.outputVolume
    }
    
    enum AudioError: Error {
        case silentFileNotFound
        case sessionSetupFailed
    }
}

import MediaPlayer

extension MPVolumeView {
    static func setVolume(_ volume: Float) {
        let volumeView = MPVolumeView()
        let slider = volumeView.subviews.first(where: { $0 is UISlider }) as? UISlider
        DispatchQueue.main.asyncAfter(deadline: .now() + 0.01) {
            slider?.value = volume
            volumeView.removeFromSuperview()
        }
    }
}
```

##### 7. 自定义通知音量（预渲染方案）

```swift
import AVFoundation
import UserNotifications

final class NotificationSoundManager {
    
    static let shared = NotificationSoundManager()
    
    private let fileManager = FileManager.default
    
    enum PresetVolumeLevel: Float, CaseIterable {
        case whisper = 0.15
        case soft = 0.3
        case normal = 0.5
        case loud = 0.75
        case maximum = 1.0
        
        var displayName: String {
            switch self {
            case .whisper: return "Whisper"
            case .soft: return "Soft"
            case .normal: return "Normal"
            case .loud: return "Loud"
            case .maximum: return "Maximum"
            }
        }
    }
    
    func generateCustomSound(
        baseFileName: String,
        volumeLevel: PresetVolumeLevel,
        outputName: String
    ) throws -> URL {
        guard let baseURL = Bundle.main.url(forResource: baseFileName, withExtension: "caf") else {
            throw SoundError.sourceFileNotFound
        }
        
        let outputURL = soundsDirectory.appendingPathComponent("\(outputName)_\(volumeLevel.rawValue).caf")
        
        if fileManager.fileExists(atPath: outputURL.path) {
            return outputURL
        }
        
        let inputFile = try AVAudioFile(forReading: baseURL)
        let format = inputFile.processingFormat
        let frameCount = AVAudioFrameCount(inputFile.length)
        
        guard let buffer = AVAudioPCMBuffer(pcmFormat: format, frameCapacity: frameCount) else {
            throw SoundError.bufferCreationFailed
        }
        
        try inputFile.read(into: buffer)
        
        applyVolumeGain(to: buffer, gain: volumeLevel.rawValue)
        
        let outputFile = try AVAudioFile(
            forWriting: outputURL,
            settings: format.settings
        )
        
        try outputFile.write(from: buffer)
        
        return outputURL
    }
    
    private func applyVolumeGain(to buffer: AVAudioPCMBuffer, gain: Float) {
        guard let channelData = buffer.floatChannelData else { return }
        let frameLength = Int(buffer.frameLength)
        
        for channel in 0..<Int(buffer.format.channelCount) {
            for frame in 0..<frameLength {
                channelData[channel][frame] *= gain
            }
        }
    }
    
    private var soundsDirectory: URL {
        let dir = fileManager.urls(for: .libraryDirectory, in: .userDomainMask)[0]
            .appendingPathComponent("Sounds")
        try? fileManager.createDirectory(at: dir, withIntermediateDirectories: true)
        return dir
    }
    
    func scheduleNotification(
        title: String,
        body: String,
        soundVolume: PresetVolumeLevel,
        at date: Date
    ) {
        let content = UNMutableNotificationContent()
        content.title = title
        content.body = body
        
        let soundFileName = "alert_\(soundVolume.rawValue)"
        content.sound = UNNotificationSound(named: UNNotificationSoundName("\(soundFileName).caf"))
        
        let trigger = UNCalendarNotificationTrigger(
            dateMatching: Calendar.current.dateComponents(
                [.year, .month, .day, .hour, .minute, .second],
                from: date
            ),
            repeats: false
        )
        
        let request = UNNotificationRequest(
            identifier: UUID().uuidString,
            content: content,
            trigger: trigger
        )
        
        UNUserNotificationCenter.current().add(request)
    }
    
    enum SoundError: Error {
        case sourceFileNotFound
        case bufferCreationFailed
        case outputFailed
    }
}
```

##### 8. Profile管理器

```swift
import SwiftData
import SwiftUI

@MainActor
final class ProfileManager: ObservableObject {
    
    static let shared = ProfileManager()
    
    @Published var activeProfile: VolumeProfile?
    @Published var profiles: [VolumeProfile] = []
    
    private var modelContext: ModelContext?
    
    func configure(modelContext: ModelContext) {
        self.modelContext = modelContext
        loadProfiles()
    }
    
    func loadProfiles() {
        guard let modelContext else { return }
        let descriptor = FetchDescriptor<VolumeProfile>(sortBy: [SortDescriptor(\.order)])
        profiles = (try? modelContext.fetch(descriptor)) ?? []
        
        if profiles.isEmpty {
            createDefaultProfiles()
        }
        
        activeProfile = profiles.first { $0.isDefault } ?? profiles.first
    }
    
    private func createDefaultProfiles() {
        let defaults: [(String, String, Float, Float, Bool)] = [
            ("Home", "house.fill", 0.8, 0.6, true),
            ("Office", "building.2.fill", 0.3, 0.2, false),
            ("Sleep", "bed.double.fill", 0.1, 0.0, false),
            ("Outdoor", "sun.max.fill", 1.0, 0.8, false),
        ]
        
        for (index, item) in defaults.enumerated() {
            let profile = VolumeProfile(
                name: item.0,
                iconName: item.1,
                ringerVolume: item.2,
                mediaVolume: item.3,
                isDefault: item.4
            )
            profile.order = index
            modelContext?.insert(profile)
            profiles.append(profile)
        }
        
        try? modelContext?.save()
    }
    
    func switchProfile(to profile: VolumeProfile) {
        activeProfile = profile
        AudioEngineManager.shared.applyProfile(profile)
        startLiveActivity(profile: profile)
    }
    
    func addProfile(_ profile: VolumeProfile) {
        profile.order = profiles.count
        modelContext?.insert(profile)
        profiles.append(profile)
        try? modelContext?.save()
    }
    
    func deleteProfile(_ profile: VolumeProfile) {
        guard !profile.isDefault else { return }
        profiles.removeAll { $0.id == profile.id }
        modelContext?.delete(profile)
        try? modelContext?.save()
    }
    
    private func startLiveActivity(profile: VolumeProfile) {
        let attributes = VolumeProfileAttributes(defaultProfile: profile.name)
        let state = VolumeProfileAttributes.ContentState(
            profileName: profile.name,
            ringerVolume: profile.ringerVolume,
            mediaVolume: profile.mediaVolume,
            isMuted: profile.ringerVolume == 0
        )
        
        do {
            let activity = try Activity.request(
                attributes: attributes,
                content: .init(state: state, staleDate: nil)
            )
        } catch {
            print("Failed to start Live Activity: \(error)")
        }
    }
}
```

## 📝 编写规则与规范

### 代码规范

```swift
// 1. 命名规范
// - 类型: PascalCase (VolumeProfile, AudioEngineManager)
// - 函数/变量: camelCase (applyProfile, currentVolume)
// - 常量: PascalCase或camelCase取决于作用域
// - 文件: 与主类型同名 (VolumeProfile.swift)

// 2. SwiftData模型规范
// - 所有模型必须标记 @Model
// - 必须有唯一 id: UUID
// - 关系必须明确 deleteRule
// - 不使用可选类型作为必需字段

// 3. SwiftUI视图规范
// - View结构体不超过150行
// - 提取子视图保持简洁
// - 使用 @Environment 自动注入依赖
// - 颜色使用 Asset Catalog，不硬编码

// 4. 错误处理规范
// - 使用 typed throws (Swift 6+)
// - 自定义Error枚举
// - 用户可见错误使用Alert展示
// - 开发错误使用assertionFailure

// 5. 音频处理规范
// - 所有音频操作在主线程检查后异步执行
// - AVAudioSession设置后必须try? setActive(false)
// - 音频文件格式统一使用.caf (Core Audio Format)
// - 自定义音量文件不超过30秒

// 6. 权限处理规范
// - 首次请求前必须解释原因
// - 拒绝后提供手动引导
// - 不在App启动时一次性请求所有权限
```

## 🎯 关键实现注意事项

### iOS系统限制与应对策略

| 限制 | 影响 | 应对策略 |
|------|------|----------|
| 第三方App无法直接修改系统通知音量 | 无法真正"分离"通知和铃声音量 | 通过Volume Profile快速切换 + Shortcuts自动化实现等效体验 |
| MPVolumeView滑块需要UI渲染 | 无法纯代码设置Media音量 | 使用隐藏MPVolumeView + 程序化触发slider |
| 通知音量由系统控制 | UNNotificationSound无法指定音量 | 预渲染不同音量级别的.caf文件 |
| Critical Alert需要Apple特殊授权 | 普通App无法绕过静音开关 | 使用AVAudioSession .playback类别在App前台播放 |
| 后台运行时间有限 | 无法持续监控音量 | 使用BGAppRefreshTask + Significant Location Change |

### App Store审核注意事项

1. **不要声称可以"独立控制系统音量"** — 这是不可能的，会被拒
2. **描述为"Volume Profile Manager"** — 管理和切换音量配置
3. **不要使用私有API** — 所有音量操作通过公开API实现
4. **Background Audio模式必须有实际音频用途** — 使用合法的音频播放
5. **Shortcuts集成必须提供真实价值** — 不能只是打开App

## 🔬 技术可行性验证

### 核心技术验证结果

| 技术点 | 可行性 | 验证方法 | 备注 |
|--------|--------|----------|------|
| AVAudioSession设置Ringer音量 | ✅ 可行 | .playback category + volume设置 | 需要短暂播放音频 |
| MPVolumeView设置Media音量 | ✅ 可行 | 隐藏MPVolumeView + slider程序化设置 | Apple允许的方式 |
| UNNotificationSound自定义音量 | ✅ 可行 | 预渲染不同音量级别的.caf文件 | 文件需<30秒 |
| AppIntents/Shortcuts集成 | ✅ 可行 | AppIntents框架 | iOS 16+ |
| FocusFilterIntent集成 | ✅ 可行 | FocusFilterIntent协议 | iOS 16+ |
| Control Center组件 | ✅ 可行 | ControlWidget协议 | iOS 18+ |
| Live Activity | ✅ 可行 | ActivityKit框架 | iOS 16.1+ |
| Dynamic Island | ✅ 可行 | ActivityKit + widget | iPhone 14 Pro+ |
| Per-App前台检测 | ⚠️ 有限 | 只能检测自己App的前台状态 | 需要用户手动配置规则 |
| 直接修改系统通知音量 | ❌ 不可行 | iOS沙盒限制 | 无公开API |

### 最低系统要求

- **iOS 16.0** — 基础功能
- **iOS 16.1** — Live Activities
- **iOS 17.0** — SwiftData, 改进的AppIntents
- **iOS 18.0** — Control Center组件

**推荐最低要求: iOS 17.0** (覆盖95%+活跃用户)
